## 오늘의 변경 사항

- `e141b12` fix: GoogleFonts 탭에서 미리보기 폰트를 onDisappear 시 unregister하여 false-positive 설치 상태 오인 버그 수정
- `47f83ef` chore: 빌드 번호를 30으로 동기화

## 문서 감사 결과

- `docs/04-font-preview.md` — **업데이트**  
  기존 섹션은 파일 기반 미리보기(`FontPreviewView`)만 다루었으나, GoogleFontRow의 미리보기 라이프사이클(등록 → 렌더링 → 해제) 및 미해제 시 false positive 발생 원인을 설명하는 **"GoogleFontRow 미리보기 (GoogleFonts 탭)"** 섹션 추가

## 테스트 감사 결과

- `FontManagerTests/FontManagerTests.swift` — **업데이트**  
  `GoogleFontRowPreviewLifecycleTests` 구조체 추가 (3개 테스트, 모두 통과):
  - `emptyInstalledFamiliesYieldsNoInstalledFonts` — preview 폰트 잔류가 없을 때 기본 동작 검증
  - `installedFamiliesStateDirectlyDrivesFilter` — `installedFamilies` 상태가 필터에 직접 영향을 주는 계약 검증 (버그 재현 → 해소 시나리오 포함)
  - `syncInstalledFamiliesDetectsSystemFonts` — `syncInstalledFamilies`가 실제 시스템 폰트만 감지하고 존재하지 않는 폰트는 제외하는지 검증

## 요약

두 커밋 중 핵심은 GoogleFontRow의 preview font unregister 버그 수정이다. 기존 문서(`docs/04-font-preview.md`)가 파일 기반 미리보기만 다루고 GoogleFontRow의 등록/해제 패턴을 누락하고 있었으므로 관련 섹션을 추가했다. 테스트는 View 레이어의 `.onDisappear` 동작을 직접 테스트할 수 없으므로, ViewModel의 `installedFamilies` 상태 계약과 `syncInstalledFamilies` 동작을 검증하는 회귀 테스트 3개를 추가했다. 빌드 번호 변경(47f83ef)은 문서·테스트 업데이트 대상이 아니다.
