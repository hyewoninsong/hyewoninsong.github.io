## 테스트 감사 결과

`GridView` 오토타일 렌더링 로직에 대한 EditMode 테스트가 없었음. 신규 추가:

**`Roguelike/Assets/Tests/EditMode/AutotileTests.cs`** (8개 테스트)

| 테스트 | 검증 내용 |
|--------|-----------|
| `SideHidden_WhenSNeighborIsElevated` | S 이웃이 Elevated이면 Side 자식 GO 없음 |
| `SideVisible_WhenSNeighborIsFloor` | S 이웃이 Floor이면 Side 자식 GO 존재 |
| `SideVisible_WhenSNeighborIsOutOfBounds` | 맵 경계(S 없음)이면 Side 자식 GO 존재 |
| `SingleTopFace_WhenNoCompositeRule` | composite 규칙 없으면 "Top" 단독 GO |
| `CompositeTopFace_WhenRuleHasTopSpriteRight` | `resultTopSpriteRight != null`이면 "TopL"+"TopR" |
| `CompositeSideFace_WhenRuleHasSideSpriteRight` | `resultSideSpriteRight != null`이면 "SideL"+"SideR" |
| `FlatTile_CreatesNeitherSideNorTop` | Floor 타일은 Side/Top 자식 없음 |
| *(N-only 규칙 매칭)* | `NeighborCondition` `mustExist=false` 조건 포함 검증 |
